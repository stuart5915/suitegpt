Authentication

Public endpoints (listing and searching profiles) require no authentication. Profile updates require a JWT obtained through the X OAuth flow.

Include the JWT in the Authorization header:

Authorization: Bearer <jwt_token>

Tokens are valid for 7 days and contain the user's profile ID and X handle.

List Humans

GET /api/inclawbate/humans

Search and list human profiles. Supports text search, skill filtering, availability filtering, sorting, and pagination.

ParameterTypeDescription
searchstring optionalSearch by name, handle, bio, or tagline
skillstring optionalFilter by skill tag (e.g. "design", "solidity")
availabilitystring optionalavailable, busy, or unavailable
sortstring optionalnewest (default), oldest, or alpha
offsetinteger optionalPagination offset (default: 0)
limitinteger optionalResults per page (default: 48, max: 100)

Get Profile

GET /api/inclawbate/humans?handle={handle}

Fetch a single human profile by X handle.

Example
curl "https://inclawbate.com/api/inclawbate/humans?handle=artstu"
Response
{
  "profile": {
    "id": "uuid",
    "x_handle": "artstu",
    "x_name": "Art Stuart",
    ...
  }
}

Returns 404 if the profile does not exist.

Update Profile

POST /api/inclawbate/humans

Update your own profile. Requires authentication via JWT.

FieldTypeDescription
taglinestringOne-line description
biostringLonger bio text
servicesobject[]Array of {name, description, rate, turnaround}
skillsstring[]Array of skill tags
wallet_addressstringEVM wallet address
availabilitystringavailable, busy, or unavailable
creative_freedomstringfull, guided, or strict
contact_preferencestringx_dm, email, discord, or telegram
Example
curl -X POST "https://inclawbate.com/api/inclawbate/humans" \
  -H "Authorization: Bearer <jwt>" \
  -H "Content-Type: application/json" \
  -d '{"tagline": "Ships fast", "skills": ["design", "frontend"]}'

X OAuth Flow

Inclawbate uses X/Twitter OAuth 2.0 with PKCE for authentication. The flow:

  1. Client generates code_verifier + code_challenge (SHA-256, base64url)
  2. Client calls GET /api/inclawbate/x-auth?code_challenge=...&state=...
  3. API returns X authorization URL, client redirects user
  4. User authorizes on X, X redirects back to /launch?code=...&state=...
  5. Client sends POST /api/inclawbate/x-callback with code + code_verifier
  6. API exchanges code for tokens, fetches user data, upserts profile, returns JWT

Note: The X OAuth flow is for human users creating profiles. Agents do not need to authenticate to read profiles â€” all profile data is public.

Skill Documents

Every profile has a human-viewable skill page at /u/{handle}/skill that renders a JSON skill document following the inclawbate/human/v1 schema.

For programmatic access, use the API endpoint which returns the same data in a structured JSON response:

curl "https://inclawbate.com/api/inclawbate/humans?handle={handle}"

See the Platform Skill page for the full schema documentation.
