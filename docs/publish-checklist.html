üöÄ Publishing Checklist

When a developer runs /publish in Discord, here's the complete flow to get their app live on the SUITE App Store.

üìã Admin Workflow: This checklist is for SUITE admins processing publish requests from the #pending channel.

Pre-Publish Checklist

  • App passes basic testing (opens, no crashes) /test [app-name]
  • App has proper branding (icon, name, description) /info [app-name]
  • Developer has set up EAS account /eas-check [app-name]
  • App follows SUITE design guidelines /review [app-name]
  • No placeholder content or "lorem ipsum" /scan [app-name]

Publishing Flow

1 Developer Request

Developer runs /publish [app-name] in Discord. Request appears in #pending channel.

2 Review in Discord

Admin reviews the request, checks the app code, and reacts with ‚úÖ to approve or ‚ùå to reject.

3 EAS Setup (if first time)

In the app folder, run:

npx eas-cli login
npx eas-cli init
npx eas-cli update:configure

4 Publish Update

Run the EAS update command:

eas update --branch production --message "v1.0 release"

Wait for export and upload to complete. Copy the Expo link from output.

5 Add to Database

Insert the app into Supabase apps table:

INSERT INTO apps (
  name, description, category, 
  status, icon_url, expo_link, 
  developer_name
) VALUES (
  'AppName',
  'Description here',
  'category',
  'live',
  'https://example.com/icon.png',
  'expo-link-from-step-4',
  'Developer Name'
);

6 Announce

Post in Discord #announcements with the QR code and Expo link. Tag the developer!

7 Verify

Check that the app shows up on getsuite.app/apps.

Troubleshooting

  • EAS login fails: Run npx eas-cli logout then login again
  • Update fails: Check if expo-updates is installed: npx expo install expo-updates
  • App not showing: Verify the status is set to live in database
