ğŸ“Š Analyze app store trends

The AI scrapes and analyzes popular app stores to identify opportunities:

1. Fetch top 100 apps from Product Hunt, App Store, Play Store
2. Categorize by: productivity, health, finance, social, etc.
3. Extract common features and pain points from reviews
4. Identify underserved niches (high demand, low supply)
5. Score each opportunity by: market size, competition, feasibility
6. Select highest-scoring concept for next build

Data stored in: ai_research_log table

ğŸ’¡ Generate app concept

Based on research, AI generates a complete app specification:

- App name and tagline
- Core value proposition (1 sentence)
- Target user persona
- 5-7 key features (prioritized)
- Tech stack recommendation
- Monetization strategy
- Success metrics

Output: JSON spec saved to ai_app_concepts

ğŸ”„ Build iteration loop (50x)

Each iteration follows this pattern:

FOR i = 1 to 50:
  1. WRITE: Generate/update code based on spec + previous critiques
  2. TEST:  Run app, check for crashes, test core flows
  3. CRITIQUE: Self-review asking:
     - Does it match the spec?
     - Any bugs or UX issues?
     - What's missing?
     - What could be better?
  4. IMPROVE: Plan fixes for next iteration
  
  IF iteration % 10 == 0:
     Save checkpoint to Supabase

Timing: ~2-3 min per iteration, ~3 hours total

  SELECT * FROM prompts 
  WHERE status = 'pending' 
  ORDER BY created_at ASC

  - Set prompt_incoming = True (blocks background clicks)

âŒ¨ï¸ Type prompt into IDE 
1. Click at (chat_x, chat_y) to focus input
2. Wait 500ms for focus
3. Prepend PROMPT_PREFIX (anti-browser, anti-question rules)
4. Convert to ASCII-safe text
5. pyautogui.typewrite(text, interval=0.01)
6. Press Enter to send
7. Mark slot_typing[i] = False

Speed: ~100 chars/second
