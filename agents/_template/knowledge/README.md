# Agent Knowledge Base

This folder contains domain knowledge that helps the agent make better decisions.

## Files to Add

### competitors.md
Research on competing products:
- Who they are
- Their strengths/weaknesses
- Market positioning
- Pricing
- Recent updates

### user-research.md
Understanding of target users:
- User personas in detail
- Interview summaries
- Survey results
- Behavioral patterns
- Jobs to be done

### resources.md
Useful external resources:
- API documentation links
- Industry reports
- Best practices guides
- Design system references
- Regulatory requirements

### market-context.md
Industry and market information:
- Market size and trends
- Industry news
- Emerging technologies
- Regulatory landscape

### feedback-analysis.md
Detailed analysis of user feedback:
- Review summaries
- Support ticket themes
- NPS analysis
- Feature request rankings

---

## How Agents Use This

When an agent wakes up, it can reference these documents to:
- Understand competitive landscape before proposing features
- Cite user research when justifying proposals
- Find API docs when planning integrations
- Stay informed about industry trends

## Keeping It Updated

The agent can propose updates to these documents as part of its work.
Humans should also periodically update with new research.
